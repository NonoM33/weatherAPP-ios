## Fonctionnalités ##

- Affichage des conditions météorologiques actuelles et des prévisions horaires.
- Recherche et enregistrement de plusieurs emplacements.
- Prévisions pour les jours à venir avec des détails sur la température, la pluie, la vitesse du vent et plus.
- Interface utilisateur épurée et moderne, avec un thème sombre pour un confort visuel de nuit.

## Comment ça marche ##

Cette application utilise l'API OpenWeather pour récupérer les données météorologiques en temps réel. Voici comment démarrer avec l'application :

1. Cloner le dépôt.
2. Ouvrir `WeatherApp.xcodeproj` dans Xcode.
3. Entrer votre API Key OpenWeather dans le fichier `weatherService`.

<img src="https://github.com/NonoM33/weatherAPP-ios/assets/29004520/5c899240-33ad-47b2-93d8-880ae89dbdaa" width="550">

4. Exécuter sur votre simulateur ou appareil.

## Choix Techniques ##

**ADCoordinator** 
Pour la coordination des vues J'ai choisi ADCoordinator pour sa flexibilité et son approche déclarative qui simplifie la gestion des flux de navigation dans l'application. Cela me permet de maintenir une architecture claire et de faciliter les modifications et l'évolution de l'application.

**swinject** 
Pour l'injection de dépendances l'utilisation de Swinject m'as aidé à gérer l'injection de dépendances de manière élégante et efficace. Cet outil permet une configuration centralisée des dépendances, rendant le code plus modulaire, facile à tester et à maintenir. C'est un choix stratégique pour une application évolutive et robuste.

**SwiftLint** 
Pour un code propre et conforme SwiftLint joue un rôle clé dans mon processus de développement. Cet outil m'aide à respecter les conventions de codage Swift et à maintenir une base de code propre et uniforme. En appliquant automatiquement des règles de style et des meilleures pratiques, SwiftLint garantit que mon code est non seulement fonctionnel, mais aussi lisible et maintenable, ce qui est crucial pour la durabilité et l'évolutivité de notre application.
